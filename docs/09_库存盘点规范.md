# 库存盘点规范

## 1. 适用范围

本规范适用于仓储中心的日盘、周盘、月盘和专项盘点场景，目标是保证系统库存与实物库存一致，及时发现差异、提升账实准确率。

## 2. 盘点类型

- 日盘：对高频动销或高价值商品进行小范围快速盘点。
- 周盘：按库区、品类或责任班组进行滚动抽盘。
- 月盘：对重点库位和核心 SKU 做系统性核查。
- 专项盘点：在上架异常、客户投诉、库存预警或审计要求下临时开展。

## 3. 标准盘点流程

1. 盘点前冻结相关库位或 SKU 的作业，避免边操作边盘点。
2. 导出系统库存清单，按库区分配盘点任务。
3. 现场逐货位清点数量、批次、状态和有效期。
4. 对差异项进行复盘复点，确认是否存在漏扫、错放或系统未回传。
5. 形成差异报告，明确原因、责任环节和整改措施。

## 4. 差异处理原则

若发现账实不符，应优先检查最近的入库、出库、移库和退货记录。未经复核不得直接调整系统库存。确认差异原因后，由授权人员执行库存调整，并在记录中保留依据。对于连续发生差异的库区，应安排专项复盘和流程整改。

## 5. 复盘要求

盘点结束后必须召开简短复盘，关注差异类型、重复问题和改进动作。复盘重点不是追责本身，而是识别制度、培训、系统或执行层面的薄弱点，持续降低库存差异率。
