# 异常工单升级规则

## 1. 适用范围

本规则适用于物流订单在仓储、运输、分拨、配送和售后阶段产生的异常工单升级管理，包括延迟、破损、温控异常、客户投诉和系统轨迹异常等情形。

## 2. 异常等级划分

### 普通异常

适用于影响范围有限、可在一线团队内完成闭环的问题，例如短时延迟、轻微包装受损、首次投诉、少量差异件等。

### 严重异常

适用于可能造成明显客户损失、舆情风险或业务中断的问题，例如延迟超过 48 小时、冷链温度异常、货物破损导致无法交付、客户连续投诉两次以上或系统长时间无轨迹。

### 特殊异常

适用于涉及生鲜、冷链、医药、高价值商品、监管风险或重大客户订单的场景。这类异常即使当前影响尚未扩大，也应优先升级。

## 3. 升级触发条件

满足以下任一条件时，应升级异常工单：

1. 订单延迟超过 48 小时；
2. 出现温度异常、货损严重或批量异常；
3. 客户连续投诉两次以上；
4. 涉及特殊品类或重点客户；
5. 一线处理超过承诺时限仍无法闭环。

## 4. 升级流程

1. 一线人员在系统中补全订单号、异常类型、节点、已处理动作和客户诉求。
2. 将工单升级至客服主管、运营负责人或专项值班人。
3. 升级后明确责任人、预计反馈时间和下一步动作。
4. 异常关闭前补充处理结论、责任归属和复盘建议。

## 5. 管理要求

异常工单升级的核心目的是加快资源协调和风险收敛，不是简单转派。升级后仍需保持客户沟通连续性，并确保知识库、SOP 和历史案例能支持一线人员快速判断处理路径。
