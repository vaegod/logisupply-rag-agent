# 仓储入库流程

## 1. 适用范围

本流程适用于普通商品、电子产品、冷链商品在仓储中心的入库处理，覆盖预约、到货、验收、质检、上架和异常反馈等环节。对于危险品、超大件或需要特殊监管的商品，应在本流程基础上追加专项审批要求。

## 2. 入库前准备

供应商或上游仓在发货前应至少提前一天提交送仓预约，内容包括预计到仓时间、车辆信息、商品品类、SKU 数量、是否需要温控设备以及联系人信息。仓库计划人员在系统中确认月台、人员和库位资源，并将预约状态更新为“已确认”。

如遇临时加急到货，必须由仓库主管确认是否具备接货能力。未经预约的车辆原则上不得直接卸货，避免造成月台拥堵和账物不一致。

## 3. 标准入库步骤

1. 车辆到仓后在门岗登记，核验预约单号、车牌号和承运人信息。
2. 收货人员引导车辆进入指定月台，确认封签、温控记录和外包装状态。
3. 根据送货单逐项验收 SKU、数量、批次、保质期和包装完整性。
4. 质检人员对抽检商品进行外观、标签、温度或功能性检查。
5. 验收通过后在系统中生成入库单，并打印上架标签。
6. 上架人员按照库区、货位和先进先出原则完成上架。
7. 上架完成后回传实收数量，关闭本次入库任务。

## 4. 关键控制点

- 收货数量必须与预约单、送货单和系统入库单三方一致。
- 冷链商品入库时应同步记录到仓温度和卸货时长，避免温控断链。
- 商品批次、生产日期、保质期等信息必须在系统中完整录入，便于后续追溯。
- 易碎或高价值商品上架前应增加二次核验，必要时留存照片。

## 5. 异常处理

如果发现数量短少、外包装破损、标签缺失或保质期不符合要求，应先隔离商品，再登记异常收货单，并通知采购或供应商确认处理意见。未确认前不得直接上架。若同一供应商连续出现两次以上严重入库异常，应在周例会上复盘并评估是否调整供应策略。
