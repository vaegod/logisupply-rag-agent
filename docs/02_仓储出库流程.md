# 仓储出库流程

## 1. 适用范围

本流程适用于仓储中心面向直营网点、电商客户和经销商的出库作业，涵盖波次下发、拣货、复核、打包、交接和出库确认。对于冷链、贵重品和加急件，应在通用流程基础上增加专项复核要求。

## 2. 出库前置条件

销售订单、调拨单或补货单必须先完成审核，系统库存状态应为可用库存。仓库运营人员根据发运时效、承运商揽收时间和波次策略生成出库任务，并明确优先级、库区和截止时间。

如发现订单存在欠货、冻结库存或客户地址信息异常，必须先暂停出库，由客服或计划人员补齐信息后再恢复。

## 3. 标准出库步骤

1. 系统下发拣货任务，拣货员按推荐路径到对应库位取货。
2. 拣货完成后将商品带至复核台，复核员核对 SKU、数量、批次和订单号。
3. 对需要合单或拆单的订单，先按系统指令完成归并，再进入包装环节。
4. 打包人员根据商品属性选择纸箱、保温箱、防震材料或封箱方式。
5. 打印面单后再次检查收件人信息、承运商信息和箱唛标签。
6. 按承运商或线路完成交接扫描，装车后系统更新为“已出库”。

## 4. 关键控制点

- 拣货时必须遵循先进先出和批次优先规则，避免临期商品积压。
- 复核阶段至少检查 SKU、数量、订单号和面单四项核心信息。
- 冷链商品在打包阶段需要确认冰袋、温度计和保温时效要求。
- 电子产品、玻璃制品等易损商品应附加缓冲包装，并在外箱张贴警示标签。

## 5. 异常处理

如果复核发现错拣、漏拣、少件或商品受损，应立即退回异常区重新处理。若承运商临时取消揽收，需由计划人员协调改派，并同步更新预计发运时间。对因仓内操作失误造成的出库异常，应登记责任环节，用于后续绩效和流程优化。
